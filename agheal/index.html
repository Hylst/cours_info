Ce que fait AGHeal

AGHeal organise les séances sportives, les inscriptions et le suivi des adhérents. Tout est pensé pour que le coach gère son planning, ses clients et ses groupes en quelques clics.

Logique métier

  • Un adhérent s’inscrit à une séance publiée.
  • Un coach crée activités, lieux et séances.
  • Les groupes permettent d’organiser les adhérents.
  • Les remarques coach restent privées.

Conseils d’utilisation rapides

  • Crée d’abord activités et lieux pour accélérer la planification.
  • Renseigne date, horaires et capacité avant publication.
  • Utilise les remarques coach pour les précautions privées.
  • Le statut “bloqué” empêche toute inscription.

Rappels utiles

  • Mot de passe oublié : lien dédié sur la page de connexion.
  • Notifications : rappels de séances et renouvellements via les paramètres.
  • Formulaire de contact : accessible dans la page Informations.
  • Annuler une séance : changer le statut en “Annulée”.

Questions fréquentes

  • Adhérent invisible : vérifier qu’il a créé son compte et complété son profil.
  • Inscription impossible : statut actif, séance publiée et places disponibles.
  • Besoin d’aide : contacter l’équipe coach via les contacts internes.

Rôle admin

L’admin supervise l’ensemble : utilisateurs, rôles, cohérence des données, contenu d’information publique, et sécurité des accès.

Ce que l’admin pilote

  • Gestion des rôles : admin, coach, adhérent.
  • Suivi des comptes : activation, blocage, statut de règlement.
  • Configuration des messages publics (page Informations).
  • Journalisation des actions clés.

Flux métier côté admin

Création utilisateur
→
Profil auto-créé
→
Rôle adhérent par défaut
→
Attribution coach/admin si besoin

Bonnes pratiques d’administration

  • Vérifier que chaque coach possède le rôle adéquat.
  • Créer au moins une activité et un lieu par période.
  • Contrôler les groupes pour éviter les doublons.
  • Relire les contenus publics régulièrement.

Frontend

Application React : navigation, formulaires, tableaux, calendrier. Elle consomme l’API et affiche les données.

Backend

API PHP : authentification, règles d’accès, création et mise à jour des séances, clients, groupes et lieux.

Base MySQL

Stockage des profils, séances, inscriptions et rôles. Le cœur des données métier.

Règles d’accès (simplifiées)

  • Coach : gérer séances, clients, activités, lieux et groupes.

Recherche rapide

users

Comptes d’accès, email, mot de passe, timestamps de connexion.

1 ↔ 1 avec profiles (même id)

profiles

Identité, santé, statut, préférences de notifications.

Référencée par sessions, groups, user_roles

user_roles

Rôles d’accès : admin, coach, adhérent.

FK → profiles.id

session_types

Types d’activités et descriptions.

FK → locations (lieu par défaut)

locations

Lieux des séances, adresses, notes.

FK → profiles (created_by)

sessions

Séances planifiées : date, horaire, capacité, statut.

FK → session_types, locations, profiles (created_by)

registrations

Inscriptions des adhérents aux séances.

FK → sessions et profiles

groups

Groupes d’adhérents et leurs descriptions.

FK → profiles (created_by)

user_groups

Association adhérent ↔ groupe.

FK → groups et profiles, assigned_by

app_info

Paramètres de communication (infos, précisions, message spécial).

Clés uniques par type de contenu

logs

Journal des actions, utile pour l’admin.

FK → profiles (user_id)

password_resets

Réinitialisations de mot de passe.

FK → users

refresh_tokens

Gestion des tokens de session.

FK → users

Mécanismes automatiques

Lorsqu’un compte est créé, un profil est généré automatiquement et le rôle adhérent est ajouté par défaut.
v_sessions_full donne une vue enrichie des séances. v_profiles_with_roles regroupe les rôles associés à chaque profil.
