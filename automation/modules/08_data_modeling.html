A. Concepts Relationnels (SQL)

1. Tables & Records

Oubliez Excel. Une base de données n'est pas une "feuille unique". C'est une collection de Tables (Clients, Commandes, Produits).

    Primary Key (PK)
    L'ID unique d'une ligne. (ex: user_id: 42). Ne change jamais.
    Foreign Key (FK)
    La référence vers une autre table. (ex: Dans la table Commandes, j'ai customer_id: 42).

2. Les Relations

  • One-to-Many (1:N)
    Un Client a plusieurs Commandes. Une Commande appartient à un seul Client.
    C'est 90% des cas.
  • Many-to-Many (N:N)
    Un Étudiant a plusieurs Cours. Un Cours a plusieurs Étudiants.
    Nécessite une "Table de Jonction" (ex: Inscriptions).

B. La Normalisation (Sanity Check)

❌ Erreur de Débutant (Flat Data)

Table Commandes:
- ID: 1
- Client: "Jean Dupont"
- Email: "jean@email.com"  <-- DUPLICATION !
- Produit: "iPhone 15"

Si Jean change d'email, vous devez modifier 50 commandes. Risque d'erreur : 100%.

✅ Donnée Normalisée

Table Clients:
- ID: 42, Nom: "Jean", Email: "jean@new.com"

Table Commandes:
- ID: 1, client_id: 42, product: "iPhone"

On change l'email à un seul endroit. Tout se met à jour par magie (via la relation).
