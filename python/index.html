üìñ Comprendre les expressions r√©guli√®res

Les expressions r√©guli√®res (regex) sont des s√©quences de caract√®res d√©finissant un motif de recherche. Elles sont utilis√©es pour la validation (emails, t√©l√©phones), l'extraction de donn√©es, le remplacement de texte et le parsing.

En Python, le module re fournit toutes les fonctionnalit√©s regex. Les patterns sont g√©n√©ralement d√©finis avec des raw strings (r"pattern") pour √©viter les probl√®mes d'√©chappement avec les backslashes.

Les m√©thodes principales sont match() (d√©but de cha√Æne), search() (premi√®re occurrence), findall() (toutes les occurrences) et sub() (remplacement).

G√©n√©rateurs & It√©rateurs

yield, itertools, performances

üìñ Comprendre les g√©n√©rateurs

Les g√©n√©rateurs sont des fonctions qui produisent une s√©quence de valeurs paresseusement (lazy evaluation). Au lieu de retourner toutes les valeurs d'un coup, ils les "g√©n√®rent" une √† une avec yield.

L'avantage principal est la m√©moire : un g√©n√©rateur ne stocke pas toutes les valeurs en m√©moire. Id√©al pour traiter de gros fichiers ou des s√©quences infinies.

Les generator expressions sont la version compacte : (x**2 for x in range(10)) au lieu d'une list comprehension.

G√©n√©rateurs
# Fonction g√©n√©rateur
def countdown(n):
    while n > 0:
        yield n
        n -= 1

for i in countdown(5):
    print(i)  # 5, 4, 3, 2, 1

# Generator expression
squares = (x**2 for x in range(1000000))  # √©conome en m√©moire

# G√©n√©rateur infini
def infinite_counter():
    n = 0
    while True:
        yield n
        n += 1

# yield from (d√©l√©gation)
def flatten(nested):
    for item in nested:
        if isinstance(item, list):
            yield from flatten(item)
        else:
            yield item

üìñ Comprendre async/await

La programmation asynchrone permet d'ex√©cuter des op√©rations I/O (r√©seau, fichiers) de mani√®re non-bloquante. Pendant qu'une t√¢che attend une r√©ponse, d'autres t√¢ches peuvent s'ex√©cuter.

async def d√©finit une coroutine. await suspend l'ex√©cution jusqu'√† ce que le r√©sultat soit disponible. C'est id√©al pour les applications web, les APIs et le scraping.

Le module asyncio g√®re la boucle d'√©v√©nements et l'orchestration des t√¢ches concurrentes.

Async
import asyncio

# Coroutine
async def fetch_data(url):
    print(f"Fetching {url}")
    await asyncio.sleep(1)  # simule I/O
    return f"Data from {url}"

# Ex√©cution s√©quentielle
async def main():
    result1 = await fetch_data("url1")
    result2 = await fetch_data("url2")
    return [result1, result2]

# Ex√©cution parall√®le
async def main_parallel():
    tasks = [
        fetch_data("url1"),
        fetch_data("url2"),
        fetch_data("url3")
    ]
    results = await asyncio.gather(*tasks)
    return results

# Lancer
asyncio.run(main())

# Avec timeout
try:
    result = await asyncio.wait_for(fetch_data("url"), timeout=5.0)
except asyncio.TimeoutError:
    print("Timeout!")
